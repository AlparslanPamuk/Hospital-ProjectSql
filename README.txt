The aim of this program is to manage a hospital from one hand and multiple hands. For all roles authorisation is given less, medium and maximum.
In this program Secretary rules the announcements appointments and all needs of hospital. They can also set a meeting and doctors can see them.
An appointment which secretary created automatically moves into doctor's system.
Even if the user did not create an account, program allows to create a user.
An interactive interface is used to catch user's attention.
Gif and images are in the folder Bin/Debug. All of them called by code to program.

Sql connection is defined in class named "sqlconn" so sql is used as database to call easier and prevent code crowd.

Identity number and also name-surname informations are the same in the next forms, depends on who entered the system.
